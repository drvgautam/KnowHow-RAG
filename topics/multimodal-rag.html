Multi-modal RAG

Extending retrieval beyond text — images, tables, diagrams, and charts

Why text-only RAG falls short

Enterprise documents are rarely pure text. Engineering specs contain architecture diagrams. Financial reports have charts. HR policies include org chart images. Medical records contain scans. If your RAG system can only retrieve text, it misses information locked in visual content.

Three approaches to multi-modal RAG

  • Approach 1 — Convert to text: Use OCR, table extraction models, and chart-to-data tools to convert all visual content to text before indexing. Simple to implement but lossy — spatial relationships in diagrams are hard to describe textually.
  • Approach 2 — Multi-modal embeddings: Use vision-language models (CLIP, SigLIP) to embed images directly into the same vector space as text. A query like "network architecture" retrieves both text passages and relevant diagrams. The challenge: vision embeddings and text embeddings may not align well without fine-tuning.
  • Approach 3 — Vision-language generation: Retrieve images as raw content and pass them to a multi-modal LLM (GPT-4V, Claude) for interpretation. The model "reads" the image and incorporates visual information into its answer. Most accurate but highest cost per query.

Table retrieval

Tables deserve special treatment. They are the most common non-text content and the hardest to handle:

  • Serialization: Convert tables to markdown or HTML format, preserving headers and alignment.
  • Table summarization: Generate a natural language summary of each table for embedding. The summary captures what the table is about; the raw table is included in the context.
  • Header propagation: Every chunk containing table data must include the column headers and table title.

Practical recommendations

  • Start with Approach 1 (convert to text) for 80% of use cases
  • Add Approach 3 (vision LLM) for high-value documents with critical diagrams
  • Approach 2 (multi-modal embeddings) is most useful when you have a large corpus of images that need to be searchable by text queries
  • Always store the original visual content alongside any text conversion so it can be shown to the user

ColPali represents a paradigm shift: instead of extracting text from document images, it embeds the entire page image directly using vision transformers. Retrieval operates over page screenshots, eliminating the lossy text extraction step entirely.
