Reciprocal Rank Fusion (RRF)

How to merge two ranked lists from different retrieval systems

The problem

In hybrid retrieval, you run two separate searches: dense (semantic/embedding) and sparse (BM25/keyword). Each returns a ranked list. But their scores are on completely different scales — cosine similarity ranges from -1 to 1, while BM25 scores can be any positive number. You can't just average them.

The RRF formula

RRF ignores raw scores entirely and uses only rank positions:

RRF_score(doc) = sum( 1 / (k + rank_i(doc)) )
                  for each retrieval system i

where k = 60 (a constant that prevents top-ranked
documents from dominating too heavily)

A numerical example

Suppose Document A is ranked #1 by dense search and #5 by BM25:

RRF(A) = 1/(60+1) + 1/(60+5) = 0.0164 + 0.0154 = 0.0318

Document B is ranked #3 by dense search and #2 by BM25:

RRF(B) = 1/(60+3) + 1/(60+2) = 0.0159 + 0.0161 = 0.0320

Document B wins despite not being #1 in either list — it's consistently good in both.

Why RRF works well

  • Scale-invariant: Only rank positions matter, so incompatible score scales are a non-issue.
  • Rewards consistency: A document ranked well in multiple systems gets a higher fused score than one ranked #1 in one system but absent from others.
  • No tuning needed: The k=60 constant works well across most use cases. Unlike learned fusion weights, RRF requires no training data.

RRF typically outperforms either dense or sparse retrieval alone by 5-15% on standard benchmarks. The gains come from the two systems having complementary strengths: dense catches paraphrases, sparse catches exact identifiers.
