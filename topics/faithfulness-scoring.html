Faithfulness Scoring

Using NLI models to verify that generated answers are grounded in context

What is faithfulness?

A generated answer is faithful if every claim it makes is supported by the retrieved context. Faithfulness is different from correctness — an answer can be faithful to the context but wrong if the source document is wrong. But an unfaithful answer is always a system failure.

How NLI-based scoring works

Natural Language Inference (NLI) models classify whether a hypothesis is entailed by, contradicted by, or neutral to a premise. In faithfulness scoring:

  • Premise: The retrieved context
  • Hypothesis: Each sentence of the generated answer
  • Classification: entailment (supported), contradiction (wrong), or neutral (not supported)

Running it in parallel with streaming

The key insight: faithfulness scoring runs during generation, not after. As the LLM streams its response:

  • Each completed sentence is sent to the NLI model
  • The NLI model (a distilled DeBERTa variant) scores it in ~50ms
  • By the time the full response is generated, all sentences have been scored
  • The user perceives zero added latency

What to do with low scores

  • Per-sentence flagging: Highlight unfaithful sentences in the UI with a warning
  • Aggregate score: If overall faithfulness drops below a threshold, add a warning banner
  • Automatic retry: For critical applications, regenerate the answer with a stricter prompt
  • Logging: Always log faithfulness scores for monitoring and evaluation

NLI models can have false positives (marking faithful text as contradicted) especially with complex conditional statements. Calibrate your threshold on production data before enforcing hard blocks.
