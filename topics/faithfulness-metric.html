Faithfulness as a Metric

Are the generated claims actually grounded in the retrieved context?

Definition

Faithfulness measures the fraction of claims in the generated answer that are supported by the retrieved context. A faithfulness score of 1.0 means every claim is grounded; 0.0 means none are.

Faithfulness = |supported claims| / |total claims|

How to calculate it

  • Step 1 — Claim extraction: Break the generated answer into individual claims. Each factual statement is a claim. "Employees get 15 days of leave and can carry over 5 days" contains two claims.
  • Step 2 — Entailment check: For each claim, run an NLI model with the retrieved context as premise and the claim as hypothesis. If the model predicts "entailment," the claim is supported.
  • Step 3 — Aggregate: Faithfulness = number of entailed claims / total claims.

Faithfulness vs. correctness

These are different metrics:

  • Faithfulness: "Is the answer consistent with the provided context?" (internal consistency)
  • Correctness: "Is the answer actually true?" (external truth)

A RAG system can only guarantee faithfulness — it cannot guarantee the source documents are correct. But faithfulness is the metric you can measure and optimize automatically.

Benchmark targets

  • 0.95+: Excellent. The system rarely generates unsupported claims.
  • 0.85-0.95: Good. Occasional paraphrase drift or minor extrapolations.
  • Below 0.85: Problem. The system is regularly generating claims not in the context. Investigate prompt, context assembly, or retrieval quality.

LLM-as-judge faithfulness scoring (using GPT-4 or Claude to evaluate) is more accurate than NLI models but 100x more expensive and too slow for real-time scoring. Use NLI for production, LLM-as-judge for offline evaluation.
